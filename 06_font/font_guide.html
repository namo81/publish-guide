예시 : 나눔고딕

모든 브라우저(Full)

/* nanum-gothic-regular - latin */
@font-face {
	font-family: 'Nanum Gothic';
	font-style: normal;
	font-weight: 400;
	src: url('../fonts/nanum-gothic-v17-latin-regular.eot'); /* IE9 Compat Modes */
	src: local(''),
		url('../fonts/nanum-gothic-v17-latin-regular.eot?#iefix') format('embedded-opentype'), /* IE6-IE8 */
		url('../fonts/nanum-gothic-v17-latin-regular.woff2') format('woff2'), /* Super Modern Browsers */
		url('../fonts/nanum-gothic-v17-latin-regular.woff') format('woff'), /* Modern Browsers */
		url('../fonts/nanum-gothic-v17-latin-regular.ttf') format('truetype'), /* Safari, Android, iOS */
		url('../fonts/nanum-gothic-v17-latin-regular.svg#NanumGothic') format('svg'); /* Legacy iOS */
}

폰트 두께 설정

동일폰트 - 두께만 다른 폰트 설정할 경우 font-family 는 동일하게 / font-weight 로 조정합니다.

폰트 파일에 두께가 숫자로 표기되어 있을 경우 CSS 에서 동일하게 적용하며, 숫자가 아닌 두께표기만 있을 경우는 아래 표를 참고합니다.

단, 업무별 별도 설정이 있을 경우 해당 값을 사용합니다.

브라우저 기준
폰트파일 두께 명칭 Thin light (L) Regular (R) Bold (B) Extra Bold (EB)
font-weight 설정 100 200 ~ 300 400 (normal) 700 (bold) 900

추가옵션 : font-display

IE 는 전체 미지원

FOIT / FOUT 현상 관련 설정

FOIT(Flash of Invisible Text) : 웹폰트가 로드되기 전까진 화면에 텍스트 표출을 하지 않음

FOUT(Flash of Unstyled Text) : 웹폰트가 로드되기 전까진 기본 텍스트로 보여주며, 로드 후 웹폰트로 변경됨


위 2가지 방식에 대한 css 설정 옵션이 font-display 이며, 아래는 그 설정값

  • auto - 브라우저의 기본동작에 맡기는 방식이다.
  • block - FOIT 즉, 타임아웃까지 텍스트를 보여주지 않음
  • swap - 응답이 올 때까지 무한정 기다리고 그 전까진 바로 기본폰트를 보여준다. 꼭 적용해야만 하는 중요폰트일 경우에 쓸 수 있다.
  • fallback - 100ms 내외의 시간 동안만 block을 하고 기본폰트를 보여준다. 응답이 오면 해당 폰트로 swap 하지만 짧은 시간(3s)만 기다린다.
  • optional - 100ms 내외의 시간 동안만 block을 하고 기본폰트를 보여준다. 그 후에는 대체하지 않는다. => 폰트가 상관이 없을 때

fallback 설정이 foit 를 방지하며 fout 를 최소화 할수 있으나 완벽하진 않음

font-display: swap;

CSS 외에도 script 등을 통한 좀 더 세밀한 방법들이 있으나 별도 가이드 하지 않음.

이 속성은 클라이언트측 요청이 없을 경우 별도 설정하지 않아도 무방함.

추가옵션 : unicode-range

IE 는 전체 미지원

설정한 글꼴에서 특정 범위에만 다른 설정을 적용하고자 할 경우 사용

특정 점위(ex:숫자, 영문 등) 및 특정 글자에 별도 폰트 적용이 가능


자주 쓰이는 유니코드

전체 : U+0020-007E

특수문자 : U+0020-002F, U+003A-0040, U+005B-0060, U+007B-007E

영문 : U+0041-005A(대문자), U+0061-007A(소문자)

숫자 : U+0030-0039

적용예시

@font-face {
	font-family: "NotoSansKR";
	src: url("/openconcert/fonts/NotoSansKR-Regular-Hestia.woff") format("woff");
	font-style: normal;
}
	
@font-face {
	font-family: "NotoSansKR";
	src: url("/openconcert/fonts/dancingscript-webfont.woff") format("woff");
	font-style: normal;
	unicode-range: U+0041-005A, U+0061-007A;
}

font-family 의 네이밍은 동일하게 설정하며, 범위 지정할 부분의 폰트 경로 지정

위와 같이 설정 시, 한글은 notosans 로 보이며, 영문일 경우에만 dancingscript 폰트가 적용됨.

폰트 두께 설정 시 적용순서

폰트 두께(100~900) 설정 시 설정한 두께의 폰트가 없을 경우 브라우저가 자동 적용하는 두께 우선순위


설정 font-weight가 600 이상이면, 가능한 큰 font-weight 중 가장 가까운 font-weight가 적용됩니다.(만약 없다면 작은 font-weight 중 가까운 font-weight)

폰트두께 설정 순서 - 폰트두께 600 이상
두께 100 200 300 400 500 600 700 800 900
우선순위 8 7 6 5 4 - 1 2 3

설정 font-weight가 300 이하이면, 가능한 작은 font-weight 중 가장 가까운 font-weight가 적용됩니다(만약 없다면 큰 font-weight 중 가까운 font-weight)

폰트두께 설정 순서 - 폰트두께 300 이하
두께 100 200 300 400 500 600 700 800 900
우선순위 2 1 - 3 4 5 6 7 8

설정 font-weight가 400이면, 500이 적용되고 500이 없으면 300 이하일 때 규칙이 적용됩니다.
