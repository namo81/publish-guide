폴더 구조

신규 프로젝트

html / css / images / js 를 기본으로 합니다.

  • vendor : 직접 작성하지 않은 외부에서 가져온 모든 라이브러리를 포함합니다.
  • vendor-cust : vendor 폴더에서 조금의 수정이라도 있으면 별도로 분리해서 보관합니다.
  • vendor / vendor-cust 폴더는 css / js 폴더내 각각 관리하며, 각각 파일명은 플러그인 명으로 동일하게 하여 구분될 수 있도록 합니다.
    ex) slider 플러그인이 js / css 를 포함할 경우,
    js 파일명은 slider.js / 위치는 js 폴더내 vendor
    css 파일명은 slider.css / 위치는 css 폴더 내 vendor

기타 공통사항

MNS에서는 아래의 설정을 기본으로 포함 합니다.

meta

  • title : '페이지명 | 사이트명' 구조를 각 페이지 depth 에 맞추어 모두 기입 (웹 접근성 이슈)
  • author : 추후 관리를 위하여 작성자 혹은 작성회사명을 기입
  • X-UA-Compatible : IE, Chrome등의 버젼을 상위버젼으로 고정
  • viewport : 1:1 기준으로 기입
  • og(open graph) : SNS 및 메신저에서 사이트 공유 시 사이트 기본정보 표출을 위한 정보 제공

css

  • layout : 기본 reset 및 폰트 설정 / 사이트 레이아웃 설정 / 공통요소 Style 설정 (button, table, input, tabmenu ...)
  • contents : 각 페이지별 컨텐츠 설정 / 기본 요소 변형

javascript

  • 순수 javascript 사용을 지향하며, 부득이할 경우 jquery 사용 가능
  • html body 내 최하단에 script 적용
  • common : 사이트 내 공통적으로 사용되는 기능 작성
  • 특정 페이지에서만 사용되는 기능은 해당 페이지 html 하단에 작성
    (단, 3페이지 이상 사용될 경우 common 으로 이동하며, 타 페이지에 영향을 주지 않도록 each 문 혹은 조건문 처리

스타일 적용

공통으로 적용되는 UI의 경우, 전체 태그에 걸지 않고 클래스로 적용하여 처리합니다.

/* (x) */
div { margin:10px; }
select { border:1px solid red; background:blue; }

/* (o) */
.box-my { margin:10px; }
.select-my { border:1px solid red; background:blue; }

단, 태그 전체에 스타일을 작성할 경우, 추후 다른 설정에 영향을 주지 않는 범위 내에서 최소한만 작성합니다.

table { table-layout:fixed; width:100%; border-collapse:collapse; }
button { cursor:pointer; overflow:hidden; }

var $div = $( 'div' );

기타

html에서 제공하는 기본 태그를 사용하지 않고 별도의 스크립트로만 제어할 시, 반드시 기본 속성은 지원해야 합니다.
예를 들어 기본 input태그를 사용하지 않고 스크립트의 기본 태그로만 제어시, disabled / maxlength 등 기본 속성 지원되어야 합니다.

Jquery 사용 시 페이지 로드 후 실행문은 $(document).ready() 를 사용하지 않고 $(function(){}); 문을 사용합니다.

// (x)
$(document).ready(function(){
	...
});

//(o)
$(function(){
	...
});

공통 스크립트 적용되는 UI의 경우, 전체 태그에 걸지 않고 클래스로 적용하여 처리합니다.

// (x)
// select 전체 적용됨.
$( 'select' ).customSelect()

// (o)
// .user-select 클래스가 적용된 select 태그에만 적용됨.
$( '.user-select' ).customSelect()

UI 기능은 '$(function(){});' 혹은 'window.onload' 안에서 모두 처리하지 않고, 외부 함수로 분기하여 처리합니다.
(개발과정에서 ajax 연동으로 추가 적용할 경우가 있습니다.)

// (x)
// 모든 UI 기능을 $(function(){}); 혹은 window.onload 안에서 처리.
$( function(){
	...
	...
	...
});

// (o)
// UI 기능을 함수로 분기 후, $(function(){}); 혹은 window.onload 안에서 처리.
function customSelect(){
	...
}

$( function(){
	customSelect();
});
