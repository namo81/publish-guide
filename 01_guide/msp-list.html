퍼블 화면 리스트

개요

프로젝트 진행 시 작업된 화면에 대한 관리 및 미리보기 화면 제공을 위한 리스트를 작성해야 합니다.

파일 다운로드 및 작성방법을 참고하여 작성해주세요.

/ 미리보기

파일 내에 신규 프로젝트 기준 폴더 구조 및 화면 리스트 파일이 포함되어 있습니다.

기본구조

타이틀 / 관련 작업자 / 사이트구분 메뉴(front/mobiel/admin 등) / 리스트 관련 기능 / 리스트 로 구성됩니다.

  1. - 타이틀 : 해당 프로젝트의 이름
  2. - 관련 작업자 : 프로젝트 관련 기획/디자인/퍼블/개발자 등에 대한 정보 기입
  3. - 구분메뉴 : 프로젝트가 Front / Mobile / admin 등으로 구분될 경우 링크를 넣을 수 있는 영역
  4. - 리스트 관련 기능 : 현재 리스트 진행율 / 전체 화면 대상 텍스트 검색 / 미리보기 기능 제공
  5. - 리스트 : 실 퍼블 화면 리스트

구분메뉴 : 프로젝트 내 경로를 절대경로로 작업 진행 시 Front / Mobile / Admin 등의 구분을 포트별로 분리하고 각각의 포트 URL 작성

구분메뉴 : 프로젝트 내 경로를 상대경로로 작업 진행 시 Front / Mobile / Admin 등의 구분을 폴더로 구분하며, index 파일을 각 구분별로 생성하여 해당 index 파일 경로 작성

작성방법

  1. 1. 사이트명 / 작성일, 파트별 작업자 정보 기입
  2. 2. SB 문서 IA 기준으로 depth1 ~ depth4 정리
  3. 3. 각 페이지명에 따른 파일명과 a 태그 생성
  4. 4. 진행 전 페이지의 경우 a 태그를 주석 처리 → '진행상태'가 '대기'로 자동 변경
       진행 중인 페이지 a 태그 활성화 → '진행상태'가 '진행중'으로 자동 변경
       완료된 페이지일 경우 'end-date' 칸에 완성일자 추가 → '진행상태'가 '완료'로 자동 변경
  5. 5. 단순 텍스트 수정 제외, 화면 레이아웃 변경 등의 수정이 있을 경우 '수정 및 참고' 칸에 수정 날짜 / 수정 내역 기입
  6. * 페이지가 특정 페이지에 포함되어 있을 경우, 페이지명은 작성하되 a 버튼은 생성하지 않습니다. 단, '수정 및 참고' 항목에 포함된 페이지 안내를 표시합니다.


  • 페이지가 삭제될 경우 tr 에 'del' 클래스 추가 : 전체 텍스트 취소선 / 진행상태 '삭제' 로 변경
  • 페이지가 삭제까진 아니고, 미사용 단계일 경우 'notuse' 클래스 추가 : 전체 텍스트 색상 변경 / 배경색 추가 / 진행상태 '참고용' 으로 변경

추가 사용 방법

좌측 상단 진행율 : 전체 페이지 대비 '완료' 페이지의 수를 백분율한 값 표기됩니다.

우측 상단 'Text' 검색 : 리스트 내 링크된 작업 페이지 전체에서 input 에 입력한 단어를 검색하며, 해당 페이지가 단어를 포함할 경우 링크명이 붉은색으로 표기됩니다.

우측 상단 '미리보기' : 해당 기능 on 시킨 후 '링크' 부분의 파일명에 마우스 오버 시 해당 페이지가 좌측에 iframe 형태로 표출 됩니다.
해당 iframe 의 크기는 guide 폴더 내 guiide.css 파일 수정하여 조정 가능합니다.

기타

화면 리스트는 퍼블리싱 작업에 대하여 사내 관련 팀에 공유되는 화면으로, 항상 최신 상태를 유지하여야 합니다. 작업 후 실시간으로 관리하여 주세요.
