안드로이드 4.4 에서 Touchend 버그

상세 현상

안드로이드 4.4에서 touchend 이벤트가 발생하지 않는 현상

toochstart / touchmove 까지는 문제 없으나, touchmove 에 e.preventDefault 를 주지 않을 경우 touchend 이벤트가 발생하지 않음.

touchmove 에서 touchend 로 넘어가는 중간에 기본 스크롤이 실행되면서 touchend 를 차단(?)하는 듯함.

좌우 플리킹에서는 touchmove 에 e.preventDefault 를 줌으로써 해결 가능하나, 상하 스크롤 기능일 경우에 문제가 됨.

해결 방법

이 부분은 touch 가 아닌 기본 scroll 에 기능을 넣음으로써 방법을 찾아야함.

scroll 에 on 기능을 넣고, off 용 function 을 같이 실행시켜준 다음, off 용 function 에 setTimeout 를 통해 순차적 실행.
(scroll 에 clearTimeout 도 같이 넣어주는게 좋음 - 스크롤을 연속으로 할 경우 off 기능 정지)

but. 안드로이드 4.4 에서 scroll 에 넣은 on 기능의 동작이 굉장히 느린 경우가 있음. 그럴 경우 on 기능은 touchstart / touchmove 로 설정하고 off 기능만 scroll 으로 적용.
