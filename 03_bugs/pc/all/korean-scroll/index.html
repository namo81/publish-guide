textarea, input 에서 한글 입력 중 다른 obj 클릭 시 입력중인 요소로 이동하는 버그

상세 현상

textarea, input 등에서 한글로 입력하다가 화면 스크롤 하여 다른 버튼, input 등을 클릭했을 경우. 스크롤 위치가 기존 입력하던 textarea, input 이 보이는 위치로 강제이동하는 현상.

영문/숫자 등을 입력할 시에는 발생하지 않음.

검색해본 결과, 한글이 다른 문자와 다르게 조합을 통한 문자시스템이어서 발생한다고 함.

해결 방법

근본적인 해결책은 없음.

MS 사에서도 이 부분은 원래 이렇게 디자인 된 특성이라고 안내만 하고 있음.

스크립트로 해당 textarea, input 이외의 영역에 마우스 오버할 경우 강제 blur 시켜주는 방법뿐.

강제 blur 시켜주는 방식도 문제점은 있으므로, 별도 해결책 찾을 필요가 있어보임.

마우스 오버 실행영역이 아닌 곳에 마우스를 두고 스크롤 한 뒤, 마우스 오버 영역에 이벤트가 실행되면 입력창으로 스크롤되는 현상은 다시 발생함

+ 이를 방지하기 위해 textarea, input 이외의 모든 영역에 이벤트를 걸 경우, 입력하다가 마우스가 영역을 조금이라도 벗어나면 입력 불가
