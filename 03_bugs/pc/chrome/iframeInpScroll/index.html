iframe 내 input에 키보드로 focus 했을 경우 위치값 튀는 오류

상세 현상

iframe 내에 있는 input 를 탭키로 포커싱 할때, 해당 input 내에 유저가 입력한 값이 존재할 경우 해당 input 에 포커싱 되는 순간 iframe 전체의 scroll 이 좌상단으로 강제 이동되는 버그

위 버그 발생으로 인해 iframe 를 포함하는 영역의 상위 영역의 화면 이동 버그도 발생

(position:absolute나 fixed 를 사용후 top,left 값을 주든, margin-left 값을 주든 모두 버그 발생)

위 이미지는 이 버그가 있던 페이지 - 로그인 영역에 탭키로 포커싱 할 때 발생했었음.

!! - chrome 업데이트로 인해 현재는 발생하지 않는 듯.

해결 방법

iframe 를 감싸는 태그에

-webkit-transform: translate3d(0,0,0);

를 넣어주면 iframe 내의 scroll 이동은 확인못했지만, iframe 를 감싸는 태그에 iframe 버그의 영향이 미치진 않음.
