border 관련 - style이 dotted인 라인관련 버그

상세 현상

크롬에서 border 가 dotted 로 설정되어있을 경우
일반적인 테이블 td 에서는 문제 없으나, colspan 으로 합쳐진 td 에서는 dotted 로 설정된 라인이 solid 로 보이는 버그가 있음.

(해당 td 에 설정된 라인은 괜찮으나, 해당 td 상단에 colspan 으로 합쳐지지 않은 td 가 있을 경우 해당 td 에 현상이 발생)

위 버그는 테이블 가로가 가변적일 경우 가로 사이즈에 따라 계속 오류 td 가 변동됨.

위 문제는 크롬에서만 발생.

해결 방법

• 해결 방법

border-collspse 설정을 collapse 가 아닌 separate 로 설정하면 해당 현상 사라짐

table{ border-collapse:separate; *border-collapse:collapse; border-spacing:0; width:100%; }

BUT!! - separate 로 변경할 경우 IE7 에서 hide 되어야할 caption 부분이 공간을 차지하는 현상이 발생.

때문에 *border-collapse:collapse; 로 핵을 사용해 IE 용 설정을 추가해주어야 함.
