모바일 웹뷰 코딩시 fixed 컨텐츠 있는 경우 input 에 따른 위치오류

상세 현상

IOS 에서 발생

레이어 팝업 같이 fixed 된 요소가 있는 상태에서, input, textarea 클릭으로 키패드가 올라오면 화면이 고정되지 않고 스크롤 됨.

해당 부분은 키패드 활성화 시 webview 높이값이 줄어드는게 아니라, 키패트 높이만큼 위로 밀어 올리는 듯 함.

추가로, 화면 컨텐츠는 스크롤 되는데, input 내의 채팅 커서( | ) 는 고정되이 있으므로, 화면과 커서간의 이격이 생김. 버튼 클릭 영역도 마찬가지.

해결 방법

근본적인 해결은 불가. 현재로썬 방법이 없음. 두가지 기능 동시 사용은 배제하는게 좋음.

채팅 커서 및 버튼 클릭 영역 부분은 레이어 팝업을 fixed 가 아닌 absolute 로 설정하여 배경 화면과 같이 스크롤 되게 하는 수 밖에 없음.
