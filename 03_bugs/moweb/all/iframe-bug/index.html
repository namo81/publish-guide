iframe 전체적인 오류

상세 현상

IOS의 safari 및 Webview 에서 공통 현상

(2019년 3월 - 모바일 T월드 작업 시 발생)

모바일에서 iframe 을 사용할 때 height:100% / scrolling=yes 로만 설정하면

iframe 을 감싸는 태그가 window height:100% 값을 가지고 있지 않을 경우

Android에서 height 값을 잡지 못해 뷰잉 오류 (150px)
IOS 에서는 뷰잉은 정상 이나 iframe 내에 fixed 요소를 bottom:0 으로 설정하면 bottom 위치가 잡히지 않음
( 뷰잉만 되고 실제높이는 Android 와 동일하게 인식하는 듯 )

iframe 을 감싸는 태그가 window height:100% 값을 가지고 있을 경우

Android / IOS 둘다 뷰잉은 정상

iframe 내에서 window.height 를 호출하면 Android 는 iframe 높이를 반환하나, IOS 는 iframe 내 컨텐츠 전체 높이를 호출 함


위 부분을 해결하고, IOS 에서 iframe 를 사용할 때 스크롤 되지 않는 현상을 잡기 위해

position:fixed; width:100%; height:100%; top:0; left:0; -webkit-overflow-scrolling:touch; overflow:auto;

위 설정을 적용할 경우, iframe 내에 input / text / select 등의 요소를 복합적으로 실행하면 화면 전체 스크롤이 먹통이 되거나, 모바일 키보드 입력 시 튀는 현상이 전반적으로 발생
